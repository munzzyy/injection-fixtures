# Security

injection-fixtures ships deliberately hostile test data: images containing
known visual prompt-injection payloads, packaged as pytest fixtures so agent
defenses can be tested in CI. The payloads target the *model reading the
image* - the files themselves are ordinary, well-formed PNGs generated by the
code in this repo, and the pytest plugin just hands you paths and bytes. The
plugin never executes payload content and never talks to the network.

Two things count as vulnerabilities here. A fixture image that is hostile to
the *software* opening it - malformed in a way that crashes or exploits an
image decoder - breaks the contract that these files are safe to handle and
dangerous only to read. And a plugin bug that returns the wrong image for a
requested payload id undermines every test suite that trusts it; a CI suite
that thinks it tested against a payload it never saw is worse than no test.

## Reporting a vulnerability

Please don't open a public issue for security problems. Use GitHub's private
reporting instead:

https://github.com/munzzyy/injection-fixtures/security/advisories/new

Include what you found, how to reproduce it, and the impact you'd expect.

## Supported versions

Fixes land on the latest tagged version; there's no backport policy.
